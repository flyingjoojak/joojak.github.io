# 🚀 디스코드 실시간 서버 홍보 플랫폼, HypeCord 개발기

안녕하세요! 오늘은 개인 미니 PC(N100)를 서버로 활용해, 전 세계 디스코드 서버의 활동량을 실시간으로 추적하고 홍보해주는 하이브리드 플랫폼 **HypeCord**를 개발하고 배포한 과정을 공유하려 합니다.

단순한 봇 기능을 넘어, 웹과 연동하여 유입을 만들어내는 과정에서 겪은 기술적 도전들을 STAR 기법으로 정리해 보았습니다.

---

## 📖 프로젝트 소개: HypeCord가 뭔가요?

![하잎코드](/assets/img/Hypecord.png)

HypeCord는 디스코드 서버의 **진짜 활성도**를 보여주는 플랫폼입니다. 
- **실시간 트래킹:** 음성 채널 인원과 채팅량을 Redis로 추적합니다.
- **글래스모피즘 웹:** 세련된 UI로 서버 리스트를 탐색하고 순위를 확인합니다.
- **성장 해킹:** 봇 명령어(/추천)를 웹 유입으로 연결하여 SEO와 트래픽을 동시에 잡았습니다.

---

## 🛠 사용된 기술 스택

- **Backend:** FastAPI, Pycord (Python 3.10+), PostgreSQL, Redis
- **Frontend:** Next.js 14 (App Router), TypeScript, SWR, Tailwind CSS
- **DevOps:** PM2, Cloudflare Zero Trust Tunnel (N100 미니 PC 배포)

## 💡 주요 기술 성과 및 트러블슈팅 (STAR)

### 1. "초당 수천 건의 메시지, 어떻게 처리할까?" (Redis 활용)

![다이어그램](/assets/img/diagram.png)

- **상황:** 수백 개 서버의 모든 메시지를 DB에 넣으면 서버가 버티지 못할 것이 뻔했습니다.
- **해결:** Redis의 `Sorted Set`을 도입해 타임스탬프 기반으로 데이터를 관리하고, 30초마다 10분이 지난 데이터를 자동으로 지우는 스케줄러를 가동했습니다.
- **결과:** 저사양 미니 PC에서도 CPU와 메모리 점유율을 아주 낮게 유지하며 실시간 랭킹 정보를 제공할 수 있게 되었습니다.

### 2. "왜 윈도우 PM2는 창이 자꾸 뜰까?" (Windows 프로세스 최적화)

![pm2](/assets/img/pm2.png)

- **상황:** 윈도우 환경의 PM2는 파이썬 실행 시 빈 CMD 창이 계속 뜨고 메모리 점유율 확인이 안 되는 문제가 있었습니다.
- **해결:** Node.js의 `child_process.spawn`을 활용해 파이썬 프로세스를 캡슐화한 JS 래퍼를 제작했습니다. `windowsHide: true` 옵션으로 창을 완전히 숨겼습니다.
- **결과:** GUI를 방해하지 않는 완벽한 백그라운드 구동과 정밀한 자원 모니터링이 가능해졌습니다.

### 3. "봇 사용자들을 웹사이트로 어떻게 데려올까?" (Growth Hacking)

![추천](/assets/img/추천.png)

- **상황:** 유저들이 디스코드 내에서만 활동하니 웹사이트 지표(DAU, SEO)가 낮았습니다.
- **해결:** 봇의 `/추천` 명령어를 '웹 링크 제공' 방식으로 과감히 전환했습니다. 웹사이트에는 OAuth2 로그인 상태를 체크하여 24시간 쿨타임을 관리하는 전용 추천 위젯(버튼) 및 실시간 숫자 반영(`SWR mutate`) 로직을 구현했습니다.
- **결과:** 유저가 웹에서 로그인 후 추천을 누르는 과정에서 자연스럽게 다른 서버 리스트를 탐색하게 되었으며, 이는 사이트 트래픽 상승과 구글 SEO 지수 강화라는 폭발적인 부가 효과로 이어집니다.
